# Release Notes – v2.0.0

## Highlights

AeroSync Addon Updater is now a **dual-provider updater**. In addition to X-Updater, v2.0.0 introduces full support for **iniBuilds** — authenticate with your iniBuilds account, browse your owned products, and update them directly.

## New Features

### iniBuilds Provider Support
- **Two-stage authentication**: Shopify GraphQL login → iniBuilds device registration, with automatic token renewal
- **Product catalog**: Fetches your owned iniBuilds products and displays them in a selectable dropdown
- **Activation key extraction**: Automatically reads DRM keys from `orders[].OrderProduct.drm_key` and displays them in the UI
- **Activation key persistence**: Keys are saved in your profile so they're available immediately without re-checking
- **File download**: Checksum-validated downloads from the iniBuilds CDN with the same pause/resume/cancel controls as X-Updater

### Provider Selection
- New **Provider** dropdown in the profile form (X-Updater or iniBuilds)
- Form fields adapt automatically based on the selected provider
- Each profile stores its provider choice independently

## Changes

### UI
- Provider-specific form sections with conditional visibility
- Read-only activation key field for iniBuilds profiles
- Updated i18n labels in English and German (removed "foundation" qualifier)

### Documentation
- New: `docs/inibuilds-renderer-api-analysis.md` — complete iniManager renderer bundle analysis (endpoints, auth flow, IPC channels, Electron settings)
- Updated: User Guide, Update Engine, Data & Security, Getting Started, Troubleshooting, README
- All documentation now in English

### Internal
- New `lib/inibuilds-client.js` (~3600 lines): self-contained iniBuilds client with auth, catalog, download, and checksum logic
- Extended `lib/profile-store.js` with `inibuildsActivationKey` field
- Extended `src/renderer.js` with provider-aware form handling and activation key UI sync

## Breaking Changes

None. Existing v1.x profiles and `profiles.json` files remain fully compatible. The new `provider` and `inibuildsActivationKey` fields default gracefully when absent.

## Known Limitations

- DRM-protected product installation has not been tested end-to-end (requires DRM product ownership)
