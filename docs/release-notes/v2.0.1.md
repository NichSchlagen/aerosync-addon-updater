# Release Notes – v2.0.1


## Highlights

v2.0.1 is a bugfix and documentation release that addresses several issues discovered in the iniBuilds provider introduced in v2.0.0. Pause, resume, and cancel now work correctly during iniBuilds downloads, manifest persistence is fixed, and the documentation has been improved with provider-specific guidance.


## Bug Fixes

### iniBuilds Download Pause / Resume / Cancel

• Pause and resume now work during the iniBuilds package download phase (previously only effective during file extraction)
• Cancel no longer causes a `timeout is not defined` crash — an invalid `clearTimeout` reference in `#downloadToFile` has been removed
• Cancel is now correctly propagated through the download retry loop instead of being swallowed as a generic error

### iniBuilds Manifest Persistence

• `#saveIniBuildsManifest` wrote `files: true` instead of the actual file array due to incorrect `Array.isArray()` usage — this broke delete detection on subsequent updates

### Plan Summary File Count

• `fileCount` in the update plan summary now includes both updates and deletes (previously only counted updates)


## Changes

### UI

• Removed outdated provider label suffixes: "X-Updater (current)" / "X-Updater (aktuell)" and "iniBuilds (Vorbereitung)" — both providers now display without parenthetical qualifiers
• Product directory placeholder text in the profile form now adapts based on the selected provider (X-Plane root hint for iniBuilds)

### Documentation

• Updated User Guide and Getting Started: the iniBuilds product directory must point to the X-Plane root folder, not an aircraft subfolder, because ZIP paths start with `Aircraft/…`
• Updated AGENTS.md with iniBuilds-specific architecture invariants (#14–#18), file list, and change rules


## Breaking Changes

None. Fully compatible with v2.0.0 profiles and settings.


## Known Limitations

• DRM-protected product installation has not been tested end-to-end (requires DRM product ownership)
